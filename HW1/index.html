Основні команди Git

git config --global user.name blabla - Задати\змінити ім'я; git config --global user.email blabal@bla - Задати\змінити e-mail; git init - Ініціалізувати порожній локальний Git-репозиторій; git status - Перевірити стан; git add - Додати файл(и) в індекс; git commit - Покласти зміни в локальний репозиторій; git log - Показати історію коммітів; git checkout - Зміна гілки; git branch - Показати лише локальні гілки; git push - Покласти зміни з локального у віддалений репозиторій; git pull - Оновити дані (забрати зміни) з віддаленого репозиторію по поточній гілці; git merge - Об'єднати гілки; git clone - Клонувати репозиторій, що існує, по HTTP (на прикладі GitHub); .gitignore - цей файл знаходиться в корневому сховищі репозиторія , в ньому містяться шаблони файлів які ми задаємо для визначення необхідності ігнорування визначених файлів;

Advanced

git revert - Відміна деяких наявних коммітів; git reset -Скинути поточний HEAD до вказаного стану; git rebase - Повторно застосувати коміти поверх іншої базової підказки; git cherry-pick - Застосувати зміни, внесені деякими наявними комітами; git stash - Здача змін на час, зроблені в робочій копії, щоб застосувати їх пізніше;
